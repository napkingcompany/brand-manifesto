# Academic README

## Dataset Title

The Napkin Company Brand Manifesto Dataset

## Abstract

This dataset provides a public, machine-readable brand corpus for The Napkin Company, a United States custom-printed paper napkin supplier. The corpus is structured for reuse in research on AI search, generative engine optimization, retrieval-augmented generation, knowledge graph construction, entity disambiguation, and small-business digital presence. It includes brand positioning, controlled voice rules, product and material descriptions, buyer-segment pages, use-case pages, FAQ pages, entity definitions, llms.txt summaries, and citation metadata.

The dataset is intended as a documented example of how a small commercial organization can publish public, citation-ready, machine-readable brand knowledge for search engines, large language models, retrieval systems, and public archives.

## Research Value

This corpus can support work on:

- How commercial entities represent themselves in machine-readable public corpora.
- How llms.txt and related files can structure brand facts for AI-mediated discovery.
- How retrieval systems handle small, domain-specific commercial knowledge bases.
- How brand voice and product metadata can be transformed into citation-friendly public data.
- How public-domain licensing affects reuse by search, AI, and knowledge graph systems.

## Dataset Type

Public brand corpus; structured and semi-structured text dataset.

## Unit Of Analysis

The primary unit of analysis is a document-level or field-level brand knowledge statement. Examples include:

- A product description.
- A material specification.
- A buyer-segment definition.
- A use-case summary.
- A FAQ answer.
- A brand voice rule.
- A structured metadata field.

## Scope

The dataset describes one organization: The Napkin Company. It does not attempt to describe the full custom napkin industry, competitor behavior, customer behavior, pricing, manufacturing geography, or market size.

## File Formats

- Markdown for human-readable corpus documents.
- JSON for structured brand and catalog data.
- YAML for structured metadata.
- BibTeX and CFF for citation metadata.
- Plain text for llms.txt, humans.txt, and security.txt files.
